# Self-Improving Learner — Maximum Power Edition

Claude learns any topic to mastery through an exhaustive multi-source harvest engine
followed by a rigorous autonomous self-test → score → gap-fill → rewrite loop.
It draws from every available source type, cross-validates across them, resolves conflicts,
and iterates until a confidence threshold is reached. The final output is a complete,
publication-quality mastery package.

---

## Phase 0 — Mission Briefing & Setup

### 0a. Clarify the Mission
Before starting, confirm:
1. **Topic** — Is it clear and scoped? If ambiguous, ask one clarifying question.
2. **Depth level**: `Beginner` / `Intermediate` / `Advanced` / `Research-grade`
3. **Angle or use-case** (optional): "I need this for a job interview", "building a product", "writing an essay", "pure curiosity"
4. **Confidence threshold** — default **88%**. Mention it; let the user adjust (see `references/scoring-rubric.md` for guidance).
5. **Time vs. depth preference**: "Fast overview" (cap 3 iterations) or "No shortcuts" (run to threshold, no cap)

### 0b. Initialize Loop State
Track this throughout all phases:
```
topic: ""
level: ""
threshold: 88
iteration: 0
confidence_score: 0
source_registry: []          # all sources found, with type and quality rating
knowledge_base: ""           # [KB vN] — grows and improves each loop
qa_bank: []                  # [QA vN] — refreshed each loop
gap_log: []                  # running list of every gap ever found
gaps_open: []                # gaps not yet closed
gaps_closed: []              # gaps resolved (with which source closed them)
cross_conflicts: []          # facts where sources disagreed — tracked explicitly
```

### 0c. Tell the User
```
🚀 Self-Improving Learner activated
```

## Phase 1 — Exhaustive Multi-Source Harvest

This is the most comprehensive research phase possible. Run it fully on iteration 1.
On subsequent iterations, re-run only the source types relevant to open gaps.

### SOURCE TYPES — harvest ALL of these:

#### 🌐 1. Encyclopedic & Reference Sources
- Wikipedia main article + all linked sub-articles (use `web_fetch` on each)
- Britannica, Encyclopedia.com, or domain-specific encyclopedias
- Official documentation (if topic has one — e.g., language spec, product docs, standards body)
- Government/institutional sites (.gov, .edu, .org) where relevant
- **Queries**: `"[topic] overview"`, `"[topic] fundamentals"`, `"[topic] complete guide"`, `"[topic] wikipedia"`

#### 📰 2. News & Current Developments
- Search for recent news articles (last 1–2 years) on the topic
- Look for: breakthroughs, controversies, policy changes, new research, updated understanding
- Fetch full articles with `web_fetch`, not just snippets
- **Queries**: `"[topic] 2024 2025"`, `"[topic] latest developments"`, `"[topic] recent research"`, `"[topic] new findings"`

#### 🎓 3. Academic & Research Sources
- Search for seminal papers, systematic reviews, meta-analyses, key authors in the field
- Fetch abstracts and conclusions of key papers via `web_fetch`
- Search: preprints (arxiv.org), PubMed (medical/bio), SSRN (social sciences), JSTOR
- **Queries**: `"[topic] research paper"`, `"[topic] systematic review"`, `"[topic] site:arxiv.org"`, `"[topic] site:pubmed.ncbi.nlm.nih.gov"`, `"[topic] meta-analysis"`

#### 📚 4. Books & Long-form Educational Content
- Search for the canonical books on the topic (most cited, most recommended)
- Find book summaries, chapter outlines, key takeaways via review sites
- Look for free PDFs, open textbooks, university course materials
- **Queries**: `"best books on [topic]"`, `"[topic] textbook pdf"`, `"[topic] open courseware"`, `"[topic] MIT OCW"`, `"[topic] coursera syllabus"`, `"[topic] open textbook"`

#### 💬 5. Expert Discussions, Forums & Communities
- Reddit: `site:reddit.com "[topic]"` — high-karma posts and expert comment threads
- Stack Exchange/Overflow: canonical answers for technical topics
- Quora: conceptual and "explain like I'm X" type explanations
- HackerNews: nuanced expert takes on tech/science topics
- **Queries**: `"[topic] site:reddit.com"`, `"[topic] explained site:stackoverflow.com"`, `"[topic] site:news.ycombinator.com"`, `"[topic] site:quora.com"`, `"[topic] discussion forum"`

#### 🎥 6. Video & Lecture Transcripts
- YouTube lecture transcripts, Khan Academy notes, TED talk transcripts
- University lecture notes: MIT OpenCourseWare, Stanford Online, Harvard OpenCourseWare
- Fetch transcripts via `web_fetch` on video pages
- **Queries**: `"[topic] lecture notes pdf"`, `"[topic] transcript"`, `"[topic] site:khanacademy.org"`, `"[topic] yale lectures"`, `"[topic] mit lecture transcript"`

#### 🔬 7. Expert Blogs, Substack & Practitioner Writing
- Renowned blogs from domain practitioners and researchers writing accessibly
- Substack essays, Medium deep-dives from credentialed authors
- Think tanks, research institutes, policy papers where relevant
- **Queries**: `"[topic] expert blog"`, `"[topic] deep dive substack"`, `"[topic] practitioner guide"`, `"[topic] explained by expert"`, `"[topic] field notes"`

#### 🗺️ 8. Frameworks, Models & Conceptual Structures
- Diagrams, conceptual frameworks, mental models, taxonomies, timelines
- Comparison tables, decision trees, classification systems
- Fetch and extract the textual content behind them
- **Queries**: `"[topic] framework model"`, `"[topic] taxonomy classification"`, `"[topic] mental model"`, `"[topic] conceptual map"`, `"[topic] comparison table"`, `"[topic] decision tree"`

#### ⚔️ 9. Criticisms, Debates & Alternative Perspectives
- Actively search for people who DISAGREE with the mainstream view
- Debates in the field, minority positions, critiques of popular ideas, historical controversies
- **Queries**: `"[topic] criticism"`, `"[topic] problems limitations"`, `"[topic] controversy debate"`, `"counterargument against [topic]"`, `"[topic] misconceptions"`, `"[topic] is wrong overrated"`, `"problems with [topic]"`

#### 🔗 10. Cross-Domain Connections
- Actively look for how this topic connects to related fields
- Interdisciplinary research, surprising connections, unexpected applications
- **Queries**: `"[topic] and [adjacent field]"`, `"[topic] applications in [domain]"`, `"intersection of [topic] and [adjacent topic]"`, `"[topic] interdisciplinary"`, `"[topic] unexpected connections"`

### 1b. Source Triage & Quality Rating

After harvesting, rate every source:

| Quality | Label | Criteria |
|---------|-------|----------|
| ⭐⭐⭐ Tier 1 | PRIMARY | Peer-reviewed, official docs, original research, authoritative reference |
| ⭐⭐ Tier 2 | SECONDARY | Expert-written, well-cited, credible publisher or institution |
| ⭐ Tier 3 | TERTIARY | Community content, summaries, popular press — useful for examples/angles only |

Log every source in `source_registry`: URL/title, type, tier, one sentence on contribution.

### 1c. Cross-Source Conflict Detection

Before building the KB, scan all sources for contradictions:
- Where do Tier 1 sources disagree with each other?
- Where does popular understanding (Tier 3) contradict scholarly consensus (Tier 1)?
- Log every conflict in `cross_conflicts` with both positions and the sources behind each

These conflicts MUST be explicitly represented in the KB — never silently pick one side.

### 1d. Build KB v1 — The Master Knowledge Base

Construct a rich, structured internal document:

```
[KB v1] — [Topic] Master Knowledge Base
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. EXECUTIVE SUMMARY
   [3–5 sentences: what is this, why it matters, what its key components are]

2. CORE CONCEPTS (one subsection per major concept)
   - Precise definition
   - Mechanism / How it works (step by step if needed)
   - Key properties or variants
   - Concrete real-world example (named, specific)
   - Common misunderstanding of this concept

3. CONCEPTUAL FRAMEWORK
   [How all core concepts relate to each other — hierarchical or networked structure]

4. HISTORY & DEVELOPMENT
   [Origin, key milestones, how understanding evolved over time]

5. KEY FIGURES & SCHOOLS OF THOUGHT
   [Who built or advanced this topic, and what each contributed]

6. APPLICATIONS & USE CASES
   [Where this is used, by whom, to what effect — specific examples by domain]

7. DEBATES & OPEN QUESTIONS
   [Active controversies, unresolved questions, minority positions — with sources on both sides]

8. KNOWN LIMITATIONS & CRITICISMS
   [What this topic/approach/field genuinely struggles with or cannot handle]

9. CROSS-DOMAIN CONNECTIONS
   [How this topic intersects with other fields — with specific bridges named]

10. COMMON MISTAKES & MISCONCEPTIONS
    [From research — what people most often get wrong, and the correct understanding]

11. THINGS I'M UNCERTAIN ABOUT
    [Honest gaps — updated every iteration, shrinks toward zero]

12. SOURCE CONFLICT LOG
    [Every factual dispute found, with positions and sources — never silently resolved]
```

## Phase 2 — Deep Self-Test (every iteration)

### 2a. Generate Comprehensive Q&A Bank

Create **20–35 questions** per iteration across five levels:

| Level | Type | Example question forms |
|-------|------|----------------------|
| L1 Recall | Facts, definitions | "What is X?", "Name the 3 types of Y", "When did Z happen?" |
| L2 Understanding | Mechanisms, reasons | "Explain why X occurs", "How does A differ from B?", "What causes Z?" |
| L3 Application | Scenarios, problem-solving | "Given situation X, what would happen?", "How would you use Y to solve Z?" |
| L4 Analysis | Critique, breakdown | "What are the weaknesses of X?", "Why might Y fail in context Z?" |
| L5 Synthesis | Integration, creativity | "Connect X to Y across domains", "What would a critic of X argue?", "What does X imply about Z?" |

Also include at least **3 adversarial questions** designed to probe the edges:
- "What does the mainstream view of X get wrong?"
- "Name a case where X breaks down entirely"
- "What would falsify the core claim of X?"

Label the bank **[QA v{iteration}]**.

### 2b. Answer Every Question — Using KB Only

For each question:
- Write a full answer using ONLY the current knowledge base (no new searches here)
- Rate confidence: `HIGH / MEDIUM / LOW / UNKNOWN`
- Note if the answer draws from one source (fragile) vs. multiple sources (robust)

### 2c. Adversarial Self-Critique

After answering, apply the adversarial reviewer lens:
1. **Challenge HIGH-confidence answers** — find a counter-example or edge case that breaks it
2. **Check for circular reasoning** — is the answer just restating the question?
3. **Check for single-source fragility** — backed by only one source?
4. **Check for missing mechanism** — do you know *what* but not *why* or *how*?
5. **Check for missing example** — is every claim grounded in a real, named case?
6. **Check for recency** — could this understanding be outdated?
7. **Check for coverage bias** — did you only research mainstream views?

### 2d. Score Each Answer

Use the rubric in `references/scoring-rubric.md`. Summary:
- **STRONG (2pt)**: Complete, accurate, exemplified, multi-source-backed, mechanism explained
- **ACCEPTABLE (1pt)**: Mostly correct but thin — missing example, nuance, or second source
- **WEAK (0.5pt)**: Vague, circular, single-source, missing mechanism
- **FAILED (0pt)**: Wrong, empty, contradictory, or reveals a misconception

Calculate: `confidence_score = (total_earned / total_possible) × 100`

### 2e. Extract & Classify All Gaps

Every WEAK or FAILED answer generates a gap entry:
```
Gap #[N]
Question: [the question]
Issue: [what's missing]
Type: [MISSING_FACT | SHALLOW_UNDERSTANDING | NO_EXAMPLE | CONTRADICTORY |
       SINGLE_SOURCE | OUTDATED | MISSING_MECHANISM | COVERAGE_BIAS]
Targeted search query: "[query to fix it]"
Priority: [HIGH | MEDIUM | LOW]
```

## Phase 3 — Exhaustive Gap-Filling Research (every iteration except final)

### 3a. Triage Gaps by Priority

Sort all open gaps before searching:
- **HIGH**: Failed questions, contradictions, missing mechanisms for core concepts → attack first, always
- **MEDIUM**: Weak answers, missing examples for secondary concepts
- **LOW**: Thin coverage of peripheral subtopics → address if time permits

Never skip a HIGH gap.

### 3b. Multi-Source Gap Research

For EACH gap, search across **multiple source types** based on gap type:

```
[MISSING_FACT]
  → web_search: direct factual query
  → web_fetch: primary source (official doc, paper, reference site)
  → web_search: verify from a 2nd independent source

[SHALLOW_UNDERSTANDING]
  → web_search: "[concept] mechanism explained step by step"
  → web_fetch: best article
  → web_search: "[concept] site:reddit.com OR site:stackexchange.com"
  → web_fetch: best practitioner explanation

[NO_EXAMPLE]
  → web_search: "[concept] real world example case study"
  → web_fetch: most concrete example found
  → web_search: "[concept] in practice [domain]"

[CONTRADICTORY]
  → web_search: "[claim A] vs [claim B] comparison"
  → web_fetch: best comparative analysis
  → web_search: academic/expert resolution of the conflict
  → Document BOTH positions if still unresolved

[SINGLE_SOURCE]
  → web_search: find 2+ independent sources confirming the fact
  → If 2nd source disagrees → reclassify as CONTRADICTORY

[MISSING_MECHANISM]
  → web_search: "how does [concept] actually work"
  → web_search: "[concept] step by step process explained"
  → web_fetch: most mechanistic explanation found

[OUTDATED]
  → web_search: "[concept] current understanding 2024 2025"
  → Check: was older view superseded or merely supplemented?

[COVERAGE_BIAS]
  → web_search: "criticism of [concept]"
  → web_search: "alternative view on [concept]"
  → web_search: "minority position [topic] [concept]"
```

**Minimum search budget per gap:**
- HIGH → minimum 3 searches + 2 web_fetches
- MEDIUM → minimum 2 searches + 1 web_fetch
- LOW → minimum 1 search + 1 web_fetch

### 3c. Source Triangulation

For every new fact added to the KB:
- 2+ independent Tier 1/2 sources agree → mark `[VERIFIED]`
- Only one source → mark `[SINGLE_SOURCE — unverified]`
- Sources conflict → mark `[DISPUTED: Source A says X | Source B says Y]`

Only `[VERIFIED]` facts can earn STRONG scores in the next test round.

### 3d. KB Rewrite — Full Pass

Produce **[KB v{iteration+1}]** — fully rewritten, not just appended:

1. Correct all factual errors from self-critique
2. Fill every gap with researched, triangulated content
3. Add concrete named examples for every abstract claim
4. Strengthen SINGLE_SOURCE items with corroborating sources
5. Expand "Debates & Open Questions" with new perspectives
6. Update "Common Mistakes" with new misconceptions encountered
7. Add to "Cross-Domain Connections" if new bridges found
8. Shrink "Things I'm Uncertain About" — move resolved items to proper KB sections
9. Update `cross_conflicts` log
10. Add all new sources to `source_registry` with tiers

### 3e. KB Quality Checklist

Before closing the iteration:
- [ ] Every core concept has: definition + mechanism + real example + common mistake
- [ ] Every claim is backed by at least one named source (ideally 2+)
- [ ] No circular definitions remain
- [ ] All contradictions are documented — not silently resolved
- [ ] The framework section shows how all concepts connect
- [ ] At least one criticism/limitation of the topic is captured

### 4b. Loop Decision Logic

**CONTINUE if ANY of:**
- `confidence_score < threshold`
- Any HIGH priority gap remains open
- Score improved ≥ 3% last iteration AND still below threshold

**STOP (threshold met) if ALL of:**
- `confidence_score >= threshold` ✅
- `high_priority_gaps == 0` ✅

**STOP (with honest caveat) if:**
- `iteration >= 12` → hard cap — log all remaining gaps in mastery report
- Score improved < 1% for 2 consecutive iterations → diminishing returns — note this
- No new sources found in last round → information saturation — note this

## Core Operating Principles

**Never fake confidence.** The loop only ends when the score genuinely reaches threshold. Inflating scores defeats the entire purpose. If the score stops improving, admit it and document why.

**Exhaust every source type before concluding.** Before reporting a gap as "unfillable", verify all 10 source types were attempted. A missing fact in web articles might exist in a forum post, academic paper, or expert blog.

**Represent dissent.** Every topic has minority views, critics, and unresolved debates. These must appear in the KB and study guide — not just mainstream consensus. Knowledge that ignores dissent is fragile.

**Triangulate all claims.** A claim backed by one source is marked `[SINGLE_SOURCE]` everywhere. Two+ independent Tier 1/2 sources needed for `[VERIFIED]` status. Only verified facts earn STRONG scores.

**Show the work.** Progress updates after every iteration keep the user informed. Never leave them waiting with no information.

**Source diversity = knowledge robustness.** A KB drawn only from one source type is brittle. The source coverage map in the Knowledge Map deliverable shows exactly which types were tapped — gaps explain residual uncertainty.

**Full rewrites, not just appends.** Each iteration produces a fully rewritten KB, not just added sections. Quality and coherence improve with every pass.
